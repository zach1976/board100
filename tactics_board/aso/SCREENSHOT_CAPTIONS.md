# Screenshot Captions — 16 SKUs × 6 Shots = 96 EN Benefit Captions

> Source of truth for the **main caption** baked into each App Store screenshot PNG.
> Subtitle (AAAAAA accent line below the main caption) is left to the designer per shot.
> Layout / colors / typography: see `SCREENSHOT_SPEC.md`. Universal 6-shot order: see `ASO_MASTER.md §6`.

**Tone rules:**
- 4–7 words. Readable at thumbnail size (1290×2796 → ~200×400 in search results).
- Benefit > feature. ❌ "Tap to draw arrows." ✅ "Show every run before it happens."
- No emoji. No exclamation points. No "Best/Top/Free" filler — those live in description.
- One concrete promise per caption. Never two.

**Universal shot purposes (drive the caption choice):**

| # | Subject | Caption job |
|---|---------|-------------|
| 1 | Sport-loaded hero formation | "What this SKU does" — answer the search-result thumbnail question |
| 2 | Drawing in action | "How fast" — speed-of-planning promise |
| 3 | Animation playback (motion trails) | "The differentiator" — only this app does this |
| 4 | Formation picker | "Zero setup" — open-the-box-and-go |
| 5 | Share / export modal | "Team comms" — kills the whiteboard-photo workflow |
| 6 | Language picker / settings | "Trusted globally" — scale signal |

## Localization plan (future work)

These 96 EN captions are the seed. Per `ASO_MASTER §4`, each locale must re-conceive (not machine-translate) the captions to fit its tone-of-voice:

- **zh-Hans / zh-Hant**: quantified, direct ("15 项运动，一块板"). Avoid English jargon — translate Pick-and-Roll as 挡拆 / 擋拆.
- **th**: sabai accessibility — open with "ใครๆ ก็..." or "สบายๆ" where the EN caption is a tactical promise, to soften it for Thai market.
- **ja**: more formal, slightly indirect ("戦術が、見える。").
- **ko**: energetic-imperative ("작전을, 한 번에.").
- **es / fr / vi / id / ms**: direct translation works if the tactical concept is universal; otherwise swap the headline tactic (e.g., Roland Garros → Copa Libertadores for es-AR).

Production order suggestion: ship EN + zh-Hans + th first (highest-volume markets where this app already ranks). Add remaining locales when those three prove conversion lift in screenshot A/B.

## Pipeline tool

`tactics_board/tool/aso_caption_overlay.py` reads this file and overlays each caption onto the raw localized screenshot from `aso/screenshots_localized/<sku>/<locale>/`, producing 1290×2796 captioned PNGs in `aso/screenshots_captioned/<sku>/<locale>/`.

```bash
# one SKU:
python3 tactics_board/tool/aso_caption_overlay.py --sku soccer --locale en-US

# all 16 SKUs for EN:
python3 tactics_board/tool/aso_caption_overlay.py --all --locale en-US

# overwrite the raw localized PNGs in place (use carefully):
python3 tactics_board/tool/aso_caption_overlay.py --all --in-place
```

Current limitations of the v0 tool:
- No device mockup frame around the screenshot (flat composite)
- Falls back to Helvetica Neue Bold when SF Pro is unavailable (macOS rarely ships SF Pro as a readable .ttf)
- Caption is one accent color (white); the yellow #FFD600 accent from the spec is unused
- Single-line vs two-line auto-sizing only; no manual line-break override

For the next polish pass: add device-frame PNG asset, support `<accent>...</accent>` markup in captions for `#FFD600` styling, accept `--font-path` flag for SF Pro.
