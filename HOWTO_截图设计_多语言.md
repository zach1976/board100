# 如何把「真实 App 截图」+「设计图」合成「多语言上架截图」

> 本文讲**方法**(思路 + 步骤),工具与参数细节见 `SCREENSHOT_PIPELINE_V2.md`。
> 目标:把 app 真实运行截图,套进统一的营销设计里,批量产出 **16 运动 × 11 语言 × 6 幕 = 1056 张** App Store 截图(1290×2796)。

---

## 一、核心思路:三层叠加

每张上架截图 = **三层**叠在一起,各层来源不同、可独立替换:

```
┌─────────────────────────────┐
│  ① 设计层(chrome)          │  深蓝渐变 + 场地线背景 + 徽章
│     ┌───────────────────┐   │  + 双行白/绿标题 + 副标题 + 倒影
│     │  ② 手机外框        │   │  ← 代码绘制,所有运动语言通用
│     │   ┌─────────────┐ │   │
│     │   │ ③ 真实截图  │ │   │  ← app 跑出来的原图(随运动+语言变)
│     │   │  (app UI)   │ │   │
│     │   └─────────────┘ │   │
│     └───────────────────┘   │
└─────────────────────────────┘
```

- **③ 真实截图**:唯一"真实"的部分。每个运动每个语言都不同(连 app 内的 移动/画线/Add 等按钮文字都是本地化的)。
- **② 手机外框**:钛色边框 + 灵动岛 + 侧键 + 倒影,**代码画一次**,所有图通用。
- **① 设计层**:背景 + 徽章 + 标题文字。背景按运动画(场地线 + 运动图标水印);**标题文字按语言换**。

**关键洞察:要做多语言,只有两样东西要"按语言准备"——③ 真实截图(跑出本地化 UI)和 ① 标题文字(翻译)。其余全部复用。**

---

## 二、合成一张图的步骤(单元)

给定 `(运动, 语言, 第几幕)`:

1. **背景**:按运动画深蓝渐变 + 透视场地线 + 右上角该运动图标水印。
2. **手机**:画钛框,把"③ 真实截图"缩放贴进屏幕区,加灵动岛 + 倒影。
3. **徽章**:画 pill,放该运动图标 + 「TACTICS BOARD」。
4. **标题**:取该 `(运动,语言,幕)` 的文案 —— 第 1 行白色 + 第 2 行绿色(两侧短横),下面副标题。按语言选字体、自动缩放到不超宽。

→ 输出 1290×2796 PNG。

---

## 三、批量做"多语言"的方法(关键)

把"按语言变化的两样东西"各做成一张**可查表的数据**,合成器循环查表即可:

### A. 真实截图 → 按语言批量跑(③)
- 用集成测试 `integration_test/appstore_screenshots.dart`,**循环 11 个语言** `startLocale`,每个语言把 app 切到该语言后截 6 幕。
- 6 幕剧情固定:空场 → 站位 → 时间轴 → **Add 菜单** → 路线 → 回放。
- 产物:`aso/screenshots_v2_raw/<运动>/<语言>/s{1..6}.png`。
- 量大易偶发中断 → **按运动分批跑 + 缺图重试**(`tool/capture_v2.sh`)。

### B. 标题文案 → 一张多语言表(①)
- 英文基准:每运动 6 句 `[白色词, 绿色词, 副标题]`,**按运动适配措辞**(网类用 rally/shots,团队类用 attack/runs)。
- 翻译:把英文表丢给**每语言一个翻译代理**,各自产出 `aso/captions_loc/<语言>.json`,合并进 `aso/captions_v2.json`:
  ```
  { "<运动>": { "<语言>": [ [白, 绿, 副标题], …6… ] } }
  ```
- 翻译要点:两行是**上白下绿**堆叠,要按该语言语序拆得自然读通(日/韩是宾→谓,拆点不同);CJK/泰不要手动大写。

### C. 字体 → 按语言映射
- 拉丁(en/es/fr/id/ms/vi):Helvetica Neue Condensed Black(越南文声调也支持)。
- 简中 Hiragino W6 / 繁中 Heiti TC / 日 Hiragino / 韩 AppleSDGothic Bold / 泰 Thonburi。
- CJK 非压缩字体更宽 → 大标题**自动缩小**到不超画布宽。

### D. 一键全量
合成器 `--all` 三重循环 `运动 × 语言 × 6幕`,逐张查 B 取文案、取 A 的原图、按 C 选字体,合成并**清掉目标目录旧图**后写入 `fastlane/screenshots/<运动>/<语言>/`(上传源)。

---

## 四、几个让方法成立的要点

1. **图标用 app 自带的,不用 emoji**:从 `SportGlyph` 导出透明 PNG(`test/export_glyphs_test.dart`),徽章 + 水印统一、专业。
2. **真实器材摄影图做不出来也别硬抠**:低清参考图抠出来是碎片;统一用扁平图标即可。
3. **旗舰多运动 app**:6 幕分别取 6 个不同运动的真实截图,直观表达"一个 app 多种运动"。
4. **先出 1 张样张确认风格,再全量**(避免 1056 张做完才发现不对)。
5. **上传**:截图属于版本,审核中(IN_REVIEW)是锁定的 → 要么等下个版本,要么撤回审核换图重提(`tool/resubmit_v2.py`)。

## 五、复用清单(下次换设计/加语言)

| 要改的东西 | 改哪里 |
|---|---|
| 设计样式(背景/字色/版式) | `tool/aso_design_compositor.py` |
| 文案 | `aso/captions_v2.json`(+ `captions_loc/`) |
| 加一个语言 | 集成测试 `allLocales` + 翻译一份 `captions_loc/<新>.json` + 字体映射 |
| 加一个运动 | 跑捕获 + `SportGlyph` 自动覆盖 + 文案补该运动 |
| 重出原图 | `tool/capture_v2.sh`(记得 pubspec 临时开 `integration_test`,**跑完改回注释**) |
